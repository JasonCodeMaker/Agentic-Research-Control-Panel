02

Page Index (hierarchical model)

Each concept owns one page. Open the page that owns the next decision and link to others rather than duplicating fields.

  • plan.html — T19: hypothesis, metric, baseline, budget, seed plan, experiments list, no-change boundary.
  • implementation.html — T20 + T14: owned files, change cards, reviewer verdicts, decision adjudication.
  • results.html — T23 + T10 + T13: result gate table, claim cards, validity chips, visualizations.
  • next-action.html — T24: chosen route, considered routes, evidence paths.
  • tracker.html — single home for execution state: Resume Block, to-do, ledger tables (impl review / resource alloc / latest live check), Launch readiness card (T21/T16/T1), per-run live cards (T22/T15). The prior launch.html / live.html pages have been folded in.
  • docs/index.html — method design, contracts, code anchors, audits, references.
  • _agent/context.html — slim continuity pointer; no duplication of identity.
02

User Modules

plan

Plan

Research objective, hypothesis, gates, budgets, baselines, no-change boundaries, and executable experiment list.
tracker

Tracker

Single home for execution state: Resume Block, to-do, Launch readiness (T21/T16/T1), implementation review, resource allocation, per-run live cards (T22/T15), and latest live check.
results

Results

Result gate entries, artifact validation, primary output, supported claims, unsupported claims, and next route.
docs

Docs

Context dossier: method design, code anchors, metric/dataset/runtime contracts, audits, reviews, and references.
Evidence and resume are not separate modules. They are expressed as Continuity & Verification in Overview, with exact source paths mirrored in _agent/context.html.
Template rule: package home is Overview first. User-facing details live in Plan, Tracker, Results, and Docs. Continuity and verification context is summarized in Overview and mirrored at _agent/context.html.
