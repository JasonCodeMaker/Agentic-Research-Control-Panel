Continuity pointer

This file is a slim machine-readable pointer for future agents. It does not duplicate identity fields from the package overview; it points to them by package id and stable data-* selectors.

Canonical paths

Source package: $source_path

Runtime root: $artifact_root

HTML package: research_html/packages/$package_id/

Minimum context-loading order

  1. Open index.html for identity (problem, objective, motivation, paths) and the status strip.
  2. Open tracker.html for Resume Block, ledger tables, and to-do list.
  3. Open the stage page that owns the next decision (plan / implementation / launch / live / results / next-action).
  4. Open docs/ only for the specific source needed by the next decision.

Verification rules before changing results

  • Validate live runs from the runtime root above before recording results.
  • Treat any field rendered as unmeasured as not yet measured; do not infer.
  • Identity fields are owned by index.html; this pointer never re-lists them.
