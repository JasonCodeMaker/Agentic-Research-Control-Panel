# 目前Auto-Research-Framework的核心问题

1. 上下文污染 和 模型幻觉 导致的 欺骗和用户指令不遵从
2. 缺乏 合理的 HCI 对齐 模型 和 用户 的 Context
3. 缺乏 个性化 的项目自我学习能力


## 上下文污染 和 模型幻觉 导致的 欺骗和用户指令不遵从

解决方案：
1. 使用 结构化TypeScript 和 多Agent 减少上下文 和强制 模型运行合理Test
2. 所有的实现必须是 Test Driven 的
3. 尽量减少 Workflow/Pipeline 的复杂度

## 缺乏 合理的 HCI 对齐 模型 和 用户 的 Context

解决方案：
1. 使用 Dashboard 结构的 实时更新的 HTML
2. 使用 由用户为主导的 Task 和 Objective 系统为 Agent 进行对齐

## 缺乏 个性化 的项目自我学习能力

解决方案：
1. 使用 Rule based 自我学习框架
2. 使用 自我Reflection Loop 和 合理的 Mem 保存机制
